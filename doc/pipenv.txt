1. pipenv.nvim                                            *pipenv-pipenv.nvim*

A tiny neovim plugin for automatically switching to the correct pipenv
<https://pipenv.pypa.io/en/latest/> for the open buffer.

INSTALLATION                                             *pipenv-installation*

PACKER.NVIM <HTTPS://GITHUB.COM/WBTHOMASON/PACKER.NVIM> ~

>
    use({
        "jpfender/pipenv.nvim",
        requires = "nvim-lua/plenary.nvim",
    })
<


USAGE                                                           *pipenv-usage*

Call

>
    require("pipenv").set_pipenv()
<


to set the venv of the current buffer to the associated pipenv (according to
the output of `pipenv --venv` in the buffer’s CWD).

To switch venvs automatically whenever you open or switch to a buffer, use an
autocmd:

>
    vim.api.nvim_create_autocmd("BufEnter", {
        callback = function()
            require("pipenv").set_pipenv()
        end,
    })
<


To show the venv for the current buffer, call:

>
    require("pipenv").get_current_venv()
<


LUALINE COMPONENT ~

This plugin includes a `lualine` <https://github.com/nvim-lualine/lualine.nvim>
component called `pipenv` which echoes the current venv. Sample usage:

>
    lualine_x = { { "pipenv", icon = "" }, "encoding", "fileformat", "filetype" },
<

- pipenv <https://pipenv.pypa.io/en/latest/> if you want this plugin to do anything useful ;)
